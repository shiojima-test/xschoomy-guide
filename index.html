XSchooMy × Roblox 連携ガイド

明るさセンサーで Roblox の世界を暗くしよう ── 学生向けセットアップ手順書（Mac版）

SchooMy

このガイドでできること

スクーミー（オレンジボード）の明るさセンサーを使って、Roblox の世界の空を暗くしたり明るくしたりします。センサーを手で覆うと、Roblox の中が夜のように暗くなります。手を離すと元に戻ります。

仕組みはとてもシンプルです。スクーミーが明るさの値を出し、XSchooMy という橋渡しのプログラムがその値を Roblox に運び、Roblox がそれを受け取って空を変えます。

スクーミー明るさを測る
→
XSchooMy値を運ぶ（橋）
→
Roblox空を変える
進め方のコツ 上から順番に進めてください。とくに「APIキー」「Universe ID」「.env」のところは、はじめてだと迷いやすいので、画面をよく見ながらゆっくり進めましょう。分からなくなったら、近くの先生に聞いてください。
このガイドは Mac 用です 画面やコマンドはすべて Mac を想定しています。Windows 版は別途用意予定です。

用意するもの

この章のゴール：必要な機材とアカウントがそろっているか確認する。
  • スクーミー（オレンジボード）と、つなぐ USB ケーブル
  • 明るさセンサー（CdS）
  • Mac（このガイドは Mac の画面で説明します）
  • スクーミーIDE（ボードにプログラムを書き込むソフト）
  • Roblox のアカウントと Roblox Studio（インストール済み）
  • インターネット接続
はじめての人へ Roblox Studio は、Roblox の世界を作るための無料ソフトです。まだ入っていなければ、先に create.roblox.com からダウンロードしておきましょう。
2

XSchooMy（橋のプログラム）をダウンロードする

この章のゴール：橋のプログラム一式を Mac に置く。

明るさの値を Roblox に運ぶ「橋」のプログラムが XSchooMy です。下のボタンからダウンロードしてください。

XSchooMy 本体

明るさの値を Roblox に運ぶ橋のプログラム一式（ZIP）。中に Roblox 用のプログラム（luau）も入っています。

XSchooMy をダウンロード
  • 上の「ダウンロード」ボタンを押して、ZIP ファイルを保存する。
  • ダウンロードした ZIP をダブルクリックして展開（解凍）する。xschoomy-roblox というフォルダができます。
  • そのフォルダを、分かりやすい場所（例：デスクトップ）に置く。
    • 大事：このフォルダは消したり動かしたりしない 一度置いたら、フォルダの場所は変えないでください。あとで橋を起動するときに、この場所を使います。
    3

    Mac に Python を準備する

    この章のゴール：橋のプログラムを動かすための土台（Python とライブラリ）を入れる。

    XSchooMy は Python という言語で動いています。Mac で動かすために、Python 本体と、必要な部品（ライブラリ）を入れます。

    このあたりは先生と一緒に ターミナル（黒い画面でコマンドを打つアプリ）を使います。はじめての人は、先生に手伝ってもらいながら進めてください。「ターミナル」は、Finder の「アプリケーション → ユーティリティ」の中にあります。

    3-1Python が入っているか確認する

    「ターミナル」アプリを開いて、次を打って Enter：

    # Python のバージョンを確認
python3 --version

    Python 3.〇〇 のように数字が出れば、Python は入っています。「command not found」と出たら、Python を入れる必要があります（→ FAQ を参照、または先生へ）。

    3-2必要なライブラリを入れる

    XSchooMy を動かすには、次の3つの部品（ライブラリ）が必要です。

    ライブラリ役割
    pyserialオレンジボードとシリアル通信する
    requestsRoblox に値を送る（通信）
    python-dotenv.env ファイルから設定を読み込む

    ターミナルで次を打って Enter（3つまとめて入ります）：

    # 必要なライブラリをまとめて入れる
pip3 install pyserial requests python-dotenv
    「externally-managed-environment」というエラーが出たら 次のように --user を付けて打ち直してください： 
    pip3 install --user pyserial requests python-dotenv

    Roblox でレーシングのワールドを作る

    この章のゴール：自分のワールド（レーシング）を作って、開いておく。

    4-1Roblox Studio を開く

    Roblox Studio を起動すると、ホーム画面が出ます。左に「New Experience」、下に「Open a Template（テンプレートを開く）」が並んでいます。

    ▲ Roblox Studio のホーム画面。下に「Open a Template」のテンプレート一覧が並ぶ。

    4-2レーシングのテンプレートを選ぶ

    「Open a Template」の並びの右にある 「See All（すべて表示）」 を押すと、たくさんのテンプレートが出ます。その中から 「Racing（レーシング）」 を選んでクリックします。すると、下のような砂漠のレース場のワールドが開きます。

    ▲ レーシングのテンプレートを開いたところ。右の Explorer に ServerScriptService などが並ぶ。

    レーシングが見つからないとき テンプレート一覧を横にスクロールするか、「See All」を押して探してください。砂漠のレース場のような見た目のテンプレートです。

    4-3このワールドを保存する

    開いたら、File → Save to Roblox（Roblox に保存） で、自分のワールドとして保存しておきます。名前は分かりやすいもの（例：明るさセンサー連携テスト）にしましょう。

    なぜ保存するの？ あとで使う「Universe ID」は、Roblox に保存（公開）したワールドにだけ付く番号だからです。保存しておかないと次の手順に進めません。
    5

    APIキーを取る

    この章のゴール：自分専用のAPIキー（合言葉）を作る。これで橋が Roblox にデータを送れるようになる。

    APIキーは、橋（XSchooMy）が「自分の Roblox に値を送ってもいいよ」と許可するための、長い合言葉のようなものです。自分のキーを自分で作ります。

    APIキーは絶対に他の人に見せない APIキーは、自分の Roblox のパスワードのようなものです。人に教えたり、ネットに貼ったりしないでください。

    5-1Creator Dashboard を開く

    ブラウザで create.roblox.com/dashboard/credentials を開きます（Roblox にログインした状態で）。左のメニューから Open Cloud → API Keys を選び、「Create API Key（APIキーを作成）」 を押します。

    5-2名前をつける（①全般的な情報）

    「名前（必須）」のところに、分かりやすい名前を入れます。例：SchooMyRoblox。「詳細」は空のままで大丈夫です。

    ▲「① 全般的な情報」── APIキーに名前をつける。

    5-3アクセス権限を設定する（②アクセス権限）

    「APIシステムを選択」の欄をクリックして、memory-stores を選びます。すると操作を選ぶ欄が出るので、次の3つを追加します：

    追加する操作意味
    memory-store.queue:addデータを入れる
    memory-store.queue:dequeueデータを取り出す
    memory-store.queue:discardデータを捨てる

    欄の中の「add」「dequeue」「discard」にチェック（✓）が付けばOKです。

    ▲「② アクセス権限」── memory-store.queue の add / dequeue / discard を追加した状態。

    「Restrict by Experience」は？ オフのままで大丈夫です（特定のワールドだけに限定する設定です。今回は使いません）。

    5-4セキュリティ（③セキュリティ）

    「このAPIキーで使用できるIPアドレスを制限する」はオフのままでOK。「有効期限」は 有効期限なし にしておくと、しばらく使い続けられます。

    ▲「③ セキュリティ」── IP制限オフ・有効期限なし。下の「キーを保存して生成する」を押す。

    5-5キーを作って、コピーする

    下の 「キーを保存して生成する」 ボタンを押します。すると長い文字の列（APIキー）が表示されます。

    いますぐコピーして、安全な場所に貼り付けて保存！ このキーは、この画面を閉じると二度と全部は見られません。表示されたらすぐにコピーして、メモ帳などに貼って大切に保管してください。あとで .env に貼り付けます。
    6

    Universe ID を調べる

    この章のゴール：自分のワールドの番号（Universe ID）を調べてメモする。

    Universe ID は、自分のワールドに付いている番号です。橋がこの番号を見て「どのワールドに値を送るか」を決めます。調べ方は2つあります。どちらでもOKです。

    方法ACreator Dashboard で調べる（おすすめ）

  • ブラウザで create.roblox.com を開く。
  • 自分のワールド（さっき保存したレーシング）を探す。
  • そのワールドの 3点メニュー（…） を押す。
  • メニューから 「Copy Universe ID（Universe ID をコピー）」 を押す。これで番号がコピーされます。
  • メモ帳などに貼り付けて、メモしておく。

    • 方法BStudio で調べる

    Roblox Studio で自分のワールドを開き、下の コマンド欄（Output の下の入力欄） に次を打って Enter：

    print(game.GameId)

    Output に出た数字が Universe ID です。メモしておきましょう。

    覚えておこう Universe ID は 数字だけです（十数桁の長い数字になります）。これも次の .env で使います。
    7

    .env ファイルを作って設定を書く

    この章のゴール：APIキー・Universe ID・ポートを、橋のプログラムに教える。

    .env（ドット・イーエヌブイ）は、橋のプログラムに「あなたのAPIキーはこれ、ワールドの番号はこれ」と教えるための設定ファイルです。ここが一番つまずきやすいので、ゆっくり進めましょう。

    そもそも .env って何？ ふつうのメモ帳のようなテキストファイルです。中に「APIキー = ○○○」「ワールドの番号 = △△△」のように、設定を書いておきます。橋のプログラムは、起動するときにこのファイルを読んで、「このキーを使えばいいんだな」と分かる仕組みです。

    7-1見本ファイル（.env.example）を見つける

    ダウンロードして展開した xschoomy-roblox フォルダの中に、.env.example という見本ファイルが入っています。

    ファイルが見えないとき（重要） .env.example のように名前がドット（.）で始まるファイルは、Mac では最初から隠れて見えません。Finder でそのフォルダを開いて、キーボードで Command（⌘） + Shift + .（ピリオド） を押してください。隠れていたファイルが薄く表示されます。もう一度押すと元に戻ります。

    7-2見本をコピーして .env を作る

    見本（.env.example）はそのまま残して、それをコピーしたものを .env という名前にして使います。

  • .env.example を1つコピーする（ファイルを選んで ⌘+C → ⌘+V。同じフォルダの中に「.env.example のコピー」ができます）。
  • そのコピーの名前を .env に変える（ファイル名をクリックして、.env だけにする。.example の部分を消すイメージ）。
    • なぜコピーして使うの？ 見本（.env.example）を残しておけば、書き間違えても、もう一度コピーしてやり直せるからです。見本は「お手本」、.env が「自分用」と覚えましょう。

    7-3.env の中身を書く

    作った .env をテキストエディット（Mac標準のメモ帳アプリ）で開きます。次のように、= の右側に自分の値を入れてください。

    ROBLOX_API_KEY=（5章でコピーしたAPIキーを貼る）
UNIVERSE_ID=（6章で調べた番号を貼る）
SERIAL_PORT=（7-4で調べるポートを書く）
BAUD=9600
    書くときの注意（ここ大事！） ・= の前後に空白（スペース）を入れない。
    ・APIキーは全部を正確に貼る（途中で切れたり、前後に余計な空白が入らないように）。
    ・BAUD は 9600 のままでOK。
    ・「（〜）」のカッコや日本語の説明は消して、自分の値だけにする。

    正しく書けると、こんな感じになります（値は人それぞれ違います）：

    ROBLOX_API_KEY=AbCdEf1234567890...（長い文字列）
UNIVERSE_ID=1234567890
SERIAL_PORT=/dev/cu.usbserial-0001
BAUD=9600
    テキストエディットで開くと文字が変になるとき テキストエディットが「リッチテキスト」になっていると、うまくいかないことがあります。メニューの「フォーマット → 標準テキストにする」を選んでから保存してください。保存するときは、文字コードを UTF-8 にします。

    7-4シリアルポート（SERIAL_PORT）を調べる

    ポートは、スクーミー（ボード）が Mac のどの差し込み口につながっているかを表す名前です。ボードを USB でつないだ状態で、ターミナルに次を打って Enter：

    # つながっているシリアル機器を一覧表示
ls /dev/cu.*

    すると、こんな名前が出てきます：

    /dev/cu.usbserial-0001

    これを SERIAL_PORT に書きます。多くの場合 /dev/cu.usbserial-0001 ですが、末尾の番号が少し違うこともあります。usbserial という文字が入っているものを選べば、たいてい大丈夫です。

    いくつも出てきて迷うとき Bluetooth 機器などが一緒に出ることがあります。usbserial や usbmodem が入っているものがボードです。迷ったら、ボードを抜いた状態と挿した状態でそれぞれ ls /dev/cu.* を実行し、挿したときに増えた名前がボードのポートです。
    8

    Roblox にプログラム（luau）を入れる

    この章のゴール：Roblox 側に「空を変えるプログラム」を入れ、必要な設定をオンにする。

    8-1Studio の設定を2つオンにする

    Roblox Studio で自分のワールドを開き、File → Game Settings（または Experience Settings）→ Security を開きます。次の2つをオンにして、Save を押します。

    設定意味
    Allow HTTP RequestsRoblox が橋（Mac）と通信できるようにする
    Enable Studio Access to API ServicesStudio がデータの仕組みを使えるようにする

    8-2プログラム（Script）を入れる

    ダウンロードしたフォルダの中の roblox フォルダに、Roblox に貼るプログラム（luau ファイル）が入っています。

  • Studio の右の Explorer で ServerScriptService を探す。
  • ServerScriptService にカーソルを合わせ、右に出る 「＋」 を押し、Script を選ぶ。
  • できた Script を右クリック →Rename で、名前を SensorController にする。
  • その Script をダブルクリックで開き、中身を 全部消す（⌘+A → Delete）。
  • フォルダの中の SensorController.luau をテキストエディタで開き、全部コピーして、Studio に貼り付ける。
  • ⌘+S で保存。
    • luau の中身はさわらなくてOK コピーして貼るだけです。中身を読んだり変えたりする必要はありません（興味があれば読んでみてください）。
    9

    動かしてみる

    この章のゴール：センサーを覆うと空が暗くなることを確認する。

    9-1橋（XSchooMy）を起動する

    ボードを USB でつないだ状態で、フォルダの中の start-xschoomy.command をダブルクリックします。黒い画面（ターミナル）が開いて、次のような行が出ればOKです。

    # こんな行が出れば成功
LocalHttpServer listening on http://127.0.0.1:8000/latest
sent #1 {'sensor1': 52.38}
    「開けません」と出たら start-xschoomy.command を右クリック →「開く」を選び、確認画面でもう一度「開く」を押してください（はじめての一回だけ必要）。

    9-2Studio で Play する

    Roblox Studio の上の Play（▷） を押します。少し待つと、ゲーム画面が始まります。

    9-3センサーを覆う

  • 明るさセンサーを手で覆う。
  • 1〜2秒で、Roblox の空が暗くなる（夜のようになる）。
  • 手を離すと、空が明るく戻る。
    • うまくいったら大成功！ 現実の明るさが、Roblox の世界の明るさに変わりました。これが「センサーで世界を動かす」ということです。
    10

    うまくいかないとき

    この章のゴール：よくあるつまずきを自分で直せるようにする。
    空が変わらない

    ① 橋（XSchooMy）のターミナルで値（sent #… {'sensor1': …}）が流れていますか？ 流れていなければ橋が起動できていません。② Studio の「Allow HTTP Requests」がオンになっていますか？（8-1）③ Studio を一度止めて、もう一度 Play してみてください。

    ターミナルに「command not found: python3」と出る

    Python が入っていません。先生に相談するか、python.org から Python をインストールしてください。

    pip3 install で「externally-managed-environment」と出る

    pip3 install --user pyserial requests python-dotenv のように --user を付けて打ち直してください。

    .env ファイルが見つからない／見えない

    ドットで始まるファイルは Mac では隠れています。Finder でフォルダを開き、⌘ + Shift + .（ピリオド） を押すと表示されます（7-1）。

    橋を起動すると「ポートが見つからない」のようなエラーが出る

    ボードが USB でつながっているか確認してください。つないでから、ls /dev/cu.* でポート名をもう一度調べ、.env の SERIAL_PORT が合っているか確認します（7-4）。

    「start-xschoomy.command が開けない」

    右クリック →「開く」を選び、確認画面でもう一度「開く」を押してください。Mac は、はじめてのアプリにこの確認を出します。

    APIキーをメモし忘れた／なくした

    キーは作り直せます。5章をもう一度やって、新しいキーを作ってください。古いキーは Creator Dashboard で削除しておくと安全です。

    Universe ID が分からなくなった

    6章の方法でいつでも調べ直せます。Studio で print(game.GameId) を打つのが一番早いです。
